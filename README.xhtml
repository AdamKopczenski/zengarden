Zen Garden readme file

Zen Garden is a game that I, Adam Kopczenski, made up that uses a Zendo set. The purpose of this little Ruby project is to help me make illustrations that I'll need when I publish it. I cannot draw to save my life, so it really is just easier to write a whole coding project to make this happen.

The purpose of this code

Cabinet projection is a specific subcase of oblique projection, a simple enough set of rules that makes passable drawings of three-dimensional objects. What I'm aiming for here is a tool that'll turn a description of a board with pieces on it into an SVG image.

Types of coordinates

Because I'm playing with pieces on a grid, and polygons in space, and shapes on an SVG canvas, I'm going to wind up using the word "coordinates" to refer to three totally separate systems. Here they are:

Logical coordinates refer to locations on a Zen Garden board. Things like "There is an upright blue rectangle piece at e3."

Logical coordinates are given in the form (row, col). The first coordinate, row, is a letter that increases to the south. The second coordinate, col, is a number that increase to the east. Zen Garden is played on a 4×4 grid, but pieces can be placed on the line between two squares, so a 7×7 coordinate grid is needed for a game position. (The generator will actually work with grids of any size, of course.)
